    **SEC-01 -- Remove JWT fallback secret (2 files):**

    In `src/modules/auth/jwt.strategy.ts` (line 12):
    - Replace `process.env.JWT_SECRET ?? 'dev-secret'` with a helper that throws if missing.
    - Add a function at top of file: `function getJwtSecret(): string { const secret = process.env.JWT_SECRET; if (!secret) { throw new Error('JWT_SECRET environment variable is required'); } return secret; }`
    - Use `secretOrKey: getJwtSecret()` in the constructor super() call.

    In `src/modules/auth/auth.module.ts` (line 13):
    - Same pattern: replace `process.env.JWT_SECRET ?? 'dev-secret'` with an inline throw.
    - Use: `secret: (() => { const s = process.env.JWT_SECRET; if (!s) throw new Error('JWT_SECRET environment variable is required'); return s; })()`
    - NOTE: `loadAndValidateEnv()` in `src/common/env.ts` already validates JWT_SECRET >= 32 chars at startup (called in main.ts). The fallback in auth.module.ts is the actual vulnerability because the module initializes before main.ts validation runs. Removing the fallback ensures the app crashes immediately if JWT_SECRET is missing.

    **SEC-03 -- Remove hardcoded API URLs (3 files):**

    - To: `const API_URL = process.env.NEXT_PUBLIC_API_URL || '';`
    - Add a runtime check below: `if (!API_URL) { throw new Error('NEXT_PUBLIC_API_URL environment variable is required'); }`
    - Note: user-portal uses env var name `NEXT_PUBLIC_API_URL`.

    In `admin-console/src/lib/api/client.ts` (line 19):
    - Change `const API_BASE_URL = process.env.NEXT_PUBLIC_API_BASE_URL || 'http://72.61.52.70:3310';`
    - To: `const API_BASE_URL = process.env.NEXT_PUBLIC_API_BASE_URL || '';`
    - Add same runtime check: `if (!API_BASE_URL) { throw new Error('NEXT_PUBLIC_API_BASE_URL environment variable is required'); }`
    - Note: admin-console uses env var name `NEXT_PUBLIC_API_BASE_URL` (different from user-portal).

    In `Analytics/src/lib/api/client.ts` (line 13):
    - Same pattern as admin-console: replace hardcoded fallback with empty string + throw.
    - Note: Analytics also uses `NEXT_PUBLIC_API_BASE_URL`.
