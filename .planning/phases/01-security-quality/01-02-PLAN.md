  <action>
    The root `tsconfig.json` is currently a Next.js config (has `"plugins": [{ "name": "next" }]`, `"lib": ["dom", "dom.iterable", "esnext"]`, `"jsx": "preserve"`, includes `next-env.d.ts` and `.next/types`). This is wrong for a NestJS backend. The `tsconfig.build.json` extends it with `"rootDir": "./src"` and restricts includes to `src/**/*.ts`.

    Rewrite root `tsconfig.json` to be a proper NestJS configuration:

    ```json
    {
      "compilerOptions": {
        "target": "ES2022",
        "module": "commonjs",
        "lib": ["ES2022"],
        "outDir": "./dist",
        "rootDir": "./src",
        "strict": true,
        "esModuleInterop": true,
        "skipLibCheck": true,
        "forceConsistentCasingInFileNames": true,
        "resolveJsonModule": true,
        "declaration": true,
        "declarationMap": true,
        "sourceMap": true,
        "removeComments": true,
        "emitDecoratorMetadata": true,
        "experimentalDecorators": true,
        "incremental": true
      },
      "include": ["src/**/*.ts"],
      "exclude": ["node_modules", "dist", "test", "**/*.spec.ts"]
    }
    ```

    Key changes:
    - `module`: "esnext" -> "commonjs" (NestJS runs on Node, not bundler)
    - `moduleResolution`: removed "bundler" (defaults to "node" with commonjs)
    - `lib`: removed "dom", "dom.iterable" (backend has no DOM)
    - `jsx`: removed (backend has no JSX)
    - `plugins`: removed next plugin
    - `noEmit`: removed (backend needs to emit JS to dist/)
    - `isolatedModules`: removed (not needed without bundler)
    - Added `emitDecoratorMetadata` and `experimentalDecorators` (required by NestJS decorators)
    - Added `declaration`, `sourceMap`, `outDir`
    - `include`: changed from "**/*.ts" to "src/**/*.ts"
    - `exclude`: added test, dist, spec files

    IMPORTANT: The `tsconfig.build.json` extends this. After this change, `tsconfig.build.json` still works because it just adds `"rootDir": "./src"` (already set in root now) and restricts includes/excludes. Verify `tsconfig.build.json` doesn't need changes -- it should still work as-is since it overrides include/exclude.

    NOTE: Do NOT touch the frontend tsconfig files (user-portal/, admin-console/, Analytics/). Each frontend has its own tsconfig.json that is correct for Next.js. They are independent and not affected by the root tsconfig.
  </action>
